The summary of what I did:

  1. I used OpenAPI Generator (via maven) to generate server-side API interface and data classes. I wanted this part to be 100% autogenerated, but there was initially a problem: I used Swagger Generator, and it didn't want to add @Entity annotations. After some exploring I switched to OpenAPI Generator and found out how to use x-class-extra-annotation and x-field-extra-annotation (hello useSpringJpa: true).
  2. Created a dummy Controller class so that I could test my API.
  3. Added MySQL and Liquibase for db initialization.
  4. Created a Repository and made my Controller a functioning class.
  5. Added additional rest points to add/list counters, for the interactive page.
  6. Created an index.html for the app description and another page for the visual counter controls, to make it 2 pages.
  7. Added Bootstrap for the looks, some JS to make buttons work.
  8. Added db population in liquibase.
